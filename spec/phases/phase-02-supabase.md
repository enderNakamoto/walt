## Pre-work Notes

> Decisions locked in:
> - **Local Supabase (Docker)** — `supabase init` + `supabase start` for dev. Deploy to hosted project later via `supabase link` + `db push`.
> - **No auth** — no RLS policies, no user_id. All queries use service role key server-side.
> - **No browser client** — since there's no auth, all Supabase calls go through API routes using the service role key. No `createBrowserClient()` needed.
> - Docker Desktop must be running before `supabase start`

## Decisions Made

- Applied schema directly via `psql` instead of `supabase db push` (simpler for initial setup, no migration files needed)
- Created storage bucket via direct SQL insert to `storage.buckets` table (faster than API call)
- Single `supabase` export (server-side client) — no browser client since no auth
- `upsertSnapshot` uses select-then-insert/update pattern (Supabase doesn't support upsert with composite keys easily)
- Typed helpers export functions, not a class — keeps imports simple
